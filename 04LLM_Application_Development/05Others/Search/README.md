传统推荐系统虽能基于用户历史行为实现“千人千面”的精准推荐，但存在三大核心挑战：

1. **时效困境**：无法实时捕捉站外突发热点，导致推荐内容滞后。
2. **解码困境**：难以融合多模态数据（如图文、视频）并理解新兴热点内涵。
3. **迭代困境**：依赖人工配置热点，缺乏自动化反馈优化闭环。

技术方案：系统模拟人类专家“追热点”的流程，分为以下核心模块

1. **热点感知** 通过小时级轮询全网平台热搜榜，构建动态更新的热点知识库。
2. **热点理解** 设计基于LLM的“热点调查Agent”，通过多轮交叉验证搜索（如平台定向溯源、全网开放求证）确保信息可靠性，最终输出结构化报告（含热点详情、实体词、商业化潜力）。
3. **需求推理与素材召回** 
   - **需求推理**：LLM基于热点实体生成高转化潜力的搜索词（如“iPhone 17预约”），并通过数据驱动优化Prompt，实现自我迭代。 
   - **素材召回**：结合文本搜索、视觉搜索、内容种草等多路召回策略，匹配商品与内容。
4. **相关性机审** 采用级联式三级判别模型（R1热点电商意图判别→R2热点-Query相关性判别→R3热点-Item相关性判别），确保素材与热点强相关。 通过Prompt自动优化和RAG系统引入人类审核案例，提升判别准确性。
5. **话题聚合** 将描述同一事件的热点词条自动聚类（如“iPhone 17发布会”相关词条归并为同一话题），减少冗余，提升运营效率。
6. **人审确认** 人工专家进行价值定级、调性复核、推荐理由优化，同时将审核数据回流至系统，驱动AI模型持续进化。
7. **流量策略与退场机制** 根据热点等级（如S/A/B级）分配差异化流量，并设置动态退场规则（如短时效热点一周后退场）。
